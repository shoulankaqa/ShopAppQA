## REQ_NF_001 — Производительность

1.1. Время ответа API на типичные запросы (просмотр списка товаров, добавление в корзину, авторизация) при нормальной нагрузке и стабильном интернет-соединении должно удовлетворять следующим условиям:
- 1.1.1. Медианное время ответа (P50) — не более 1 секунды.
- 1.1.2. Время ответа для 95-го перцентиля (P95) — не более 3 секунд.

1.2. Измерения производительности проводятся:
- 1.2.1. На тестовом стенде, конфигурация которого сопоставима с целевой инфраструктурой.
- 1.2.2. С использованием реалистичных тестовых данных и сценариев нагрузки.

1.3. В требования не включаются:
- 1.3.1. Экстремальные случаи перегрузки.
- 1.3.2. Нестабильные сетевые условия (потери пакетов, высокий лаг и т.п.).

## REQ_NF_002 — Безопасность

2.1. Пароли пользователей должны храниться в виде криптографического хеша.  
2.2. Система не должна отображать пароль пользователя в открытом виде ни в одном из интерфейсов.  
2.3. Пароль не должен попадать в логи и сообщения об ошибках.  

2.4. Для продуктивной среды:
- 2.4.1. Все операции аутентификации и передачи пользовательских данных должны выполняться по защищённому протоколу HTTPS.
- 2.4.2. Использование незашифрованного HTTP в продуктивной среде не допускается.

2.5. Для сред разработки и тестирования (например, localhost:3001):
- 2.5.1. Допускается использование HTTP.
- 2.5.2. При переходе в продуктивную среду должна быть предусмотрена возможность включения HTTPS без изменений в бизнес-логике аутентификации.

## REQ_NF_003 — Кроссбраузерность

3.1. Система должна корректно работать (отображение и функциональность) в актуальных версиях следующих браузеров:
- 3.1.1. Google Chrome.
- 3.1.2. Mozilla Firefox.
- 3.1.3. Safari.
- 3.1.4. Microsoft Edge.

3.2. Под «актуальными версиями» понимаются:
- 3.2.1. Последние стабильные версии (stable release), доступные на момент релиза приложения.
- 3.2.2. Не менее одной предыдущей стабильной версии каждого из указанных браузеров.

3.3. Корректная работа должна обеспечиваться как минимум на:
- 3.3.1. Windows (Chrome, Firefox, Edge).
- 3.3.2. macOS (Chrome, Firefox, Safari).


## REQ_NF_004 — Адаптивность / Поддерживаемые устройства

4.1. Интерфейс должен корректно отображаться на desktop-устройствах с минимальным разрешением 1280×720, без критических нарушений верстки (перекрытия элементов, исчезновения критически важных кнопок, горизонтальной прокрутки при типичных сценариях использования).  
4.2. Поддержка мобильных устройств (смартфоны, планшеты) в рамках текущего проекта не требуется.
