## REQ_NF_001 — Производительность

1. Время ответа API на типичные запросы (просмотр списка товаров, добавление в корзину, авторизация) при нормальной нагрузке и стабильном интернет-соединении должно удовлетворять следующим условиям:
   - медианное время ответа (P50) — не более 1 секунды;
   - время ответа для 95-го перцентиля (P95) — не более 3 секунд.

2. Измерения производительности проводятся:
   - на тестовом стенде, конфигурация которого сопоставима с целевой инфраструктурой;
   - с использованием реалистичных тестовых данных и сценариев нагрузки.

3. В требования не включаются:
   - экстремальные случаи перегрузки;
   - нестабильные сетевые условия (потери пакетов, высокий лаг и т.п.).

## REQ_NF_002 — Безопасность

1. Пароли пользователей должны храниться в виде криптографического хеша.
2. Система не должна отображать пароль пользователя в открытом виде ни в одном из интерфейсов.
3. Пароль не должен попадать в логи и сообщения об ошибках.

### Использование защищённого канала связи (HTTPS)

4. Для продуктивной среды:
   - все операции аутентификации и передачи пользовательских данных должны выполняться по защищённому протоколу HTTPS;
   - использование незашифрованного HTTP в продуктивной среде не допускается.

5. Для сред разработки и тестирования (например, localhost:3001):
   - допускается использование HTTP;
   - при этом при переходе в продуктивную среду должна быть предусмотрена возможность включения HTTPS без изменений в бизнес-логике аутентификации.

## REQ_NF_003 — Кроссбраузерность

1. Система должна корректно работать (отображение и функциональность) в актуальных версиях следующих браузеров:
   - Google Chrome,
   - Mozilla Firefox,
   - Safari,
   - Microsoft Edge.

2. Под «актуальными версиями» понимаются:
   - последние стабильные версии (stable release), доступные на момент релиза приложения;
   - а также не менее одной предыдущей стабильной версии каждого из указанных браузеров.

3. Корректная работа должна обеспечиваться как минимум на:
   - Windows (Chrome, Firefox, Edge),
   - macOS (Chrome, Firefox, Safari).

## REQ_NF_004 — Адаптивность / Поддерживаемые устройства

1. Интерфейс должен корректно отображаться на desktop-устройствах с минимальным разрешением 1280×720, без критических нарушений верстки (перекрытия элементов, исчезновения критически важных кнопок, горизонтальной прокрутки при типичных сценариях использования).

2. Поддержка мобильных устройств (смартфоны, планшеты) в рамках текущего проекта не требуется.
