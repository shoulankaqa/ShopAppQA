## REQ_USER_001 — Просмотр пользователей (ADMIN/SELLER)

1.1. Пользователи с ролями "ADMIN" и "SELLER" должны иметь доступ к просмотру списка пользователей.  
1.2. В списке пользователей для каждой записи должны отображаться:
- 1.2.1. Идентификатор пользователя.
- 1.2.2. Логин.
- 1.2.3. Роль пользователя (например, ADMIN, SELLER, USER).

1.3. Обычный пользователь (роль USER) не должен иметь доступа к просмотру списка пользователей.

## REQ_USER_002 — Создание пользователя (ADMIN)

2.1. Пользователь с ролью "ADMIN" должен иметь возможность создавать новых пользователей.  

2.2. При создании пользователя обязательными для заполнения являются:
- 2.2.1. Логин (уникальный идентификатор учётной записи).
- 2.2.2. Пароль.
- 2.2.3. Роль пользователя (ADMIN, SELLER, USER).

2.3. Требования к паролю при создании пользователя:
- 2.3.1. По умолчанию пароль должен соответствовать требованиям REQ_AUTH_001 — Требования к регистрации.
- 2.3.2. Администратор вправе задать логин/пароль, не соответствующий REQ_AUTH_001 (например, временный/упрощённый логин/пароль), но только в административном интерфейсе при создании пользователя.

2.4. Исключение по паролю/логину для администратора:
- 2.4.1. Не отменяет обязательность уникальности логина/пароля.
- 2.4.2. Не отменяет обязательность указания роли пользователя.
- 2.4.3. Не позволяет оставлять обязательные поля пустыми.
- 2.4.4. Система должна всё равно принять и сохранить данные, операция не должна блокироваться.

2.5. При попытке создать пользователя с уже существующим логином система должна:
- 2.5.1. Не создавать пользователя.
- 2.5.2. Отобразить сообщение об ошибке о неуникальном логине.

2.6. После успешного создания пользователь должен иметь возможность авторизоваться с заданными логином и паролем.  

2.7. Доступ к операции создания пользователей должен быть ограничен только пользователями с ролью "ADMIN"; пользователи с другими ролями не должны видеть элементы управления для создания пользователя.

## REQ_USER_003 — Редактирование пользователя (ADMIN)

3.1. Пользователь с ролью "ADMIN" должен иметь возможность изменять данные профиля других пользователей.  

3.2. Доступные для редактирования поля:
- 3.2.1. Логин.
- 3.2.2. Пароль (с возможностью задать значение, в соответствии с REQ_USER_002).
- 3.2.3. Роль.

3.3. Логин пользователя:
- 3.3.1. Является изменяемым при условии проверки уникальности (не должен совпадать с логином другого пользователя).

3.4. При изменении роли или статуса пользователя изменения должны вступать в силу с момента сохранения.  

3.5. При ошибках валидации (неуникальный логин, некорректный формат данных и т.п.) система должна:
- 3.5.1. Не применять изменения.
- 3.5.2. Подсветить поля с ошибками.
- 3.5.3. Отобразить сообщения об ошибке.

## REQ_USER_004 — Удаление пользователя (ADMIN)

4.1. Пользователь с ролью "ADMIN" должен иметь возможность удалить пользователя.  

4.2. Перед удалением система должна запрашивать подтверждение действия (модальное окно с текстом подтверждения).  

4.3. После удаления пользователь не должен иметь возможность авторизоваться в системе.  

4.4. Удалённый пользователь не должен отображаться в списке пользователей.  

4.5. Если при удалении пользователя произошла ошибка, система должна:
- 4.5.1. Не изменять состояние пользователя.
- 4.5.2. Отобразить сообщение об ошибке администратору.

---

## REQ_USER_005 — Ограничение прав доступа

5.1. Пользователь с ролью "USER" не должен иметь доступа к административному функционалу (просмотр пользователей, создание, редактирование, удаление пользователей, управление ролями и т.п.).  

5.2. Попытка прямого обращения обычного пользователя к административным разделам (по URL или другим способам) должна приводить к отказу в доступе (страница ошибки/редирект).  

5.3. Доступ к функционалу управления пользователями (просмотр, создание, редактирование, удаление) должен быть доступен только пользователям с ролью "ADMIN".  

5.4. Доступ к просмотру списка пользователей может быть разрешён пользователям с ролью "SELLER" в соответствии с REQ_USER_001, при этом SELLER не должен иметь возможности изменять или удалять пользователей.
