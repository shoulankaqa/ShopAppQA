## REQ_CART_001 — Добавление товара в корзину 

1.1. Пользователь должен иметь возможность добавить товар в корзину.  
1.2. Добавление товара в корзину доступно только для товаров со статусом "В наличии".  
1.3. Попытка добавить товар со статусом "Нет в наличии" не должна приводить к добавлению товара в корзину.  
1.4. Товар можно добавить из каталога товаров по нажатию круглой кнопки с иконкой корзины напротив товара.  
1.5. После успешного добавления товара корзина должна обновляться.  
1.6. При добавлении товара с количеством больше 1 в корзину добавляется указанное количество.  
1.7. Если товар уже есть в корзине, при повторном добавлении количество товара в корзине увеличивается на добавляемое значение.    
1.8. Система не должна позволять добавить в корзину более 50 единиц одного и того же товара; при попытке превысить лимит количество ограничивается 50 и отображается сообщение об ошибке: "Максимальное количество одной позиции - 50".  
1.9. При добавлении товаров в корзину должны учитываться ограничения по итоговой стоимости корзины (см. REQ_CART_004).

## REQ_CART_002 — Удаление товара из корзины

2.1. Пользователь должен иметь возможность удалить товар из корзины.  
2.2. Итоговая стоимость должна пересчитываться после удаления товара.  
2.3. Удаление можно выполнить:
- 2.3.1. По кнопке "Убрать позицию" для отдельного товара в корзине.
- 2.3.2. По кнопке "Очистить корзину" (удаление всех товаров из корзины).
- 2.3.3. По кнопке "-" (минус) напротив товара в корзине (уменьшение количества; при попытке уменьшить количество ниже 1 товар удаляется из корзины).

2.4. После удаления товар исчезает из списка товаров в корзине.  
2.5. Если пользователь пытается удалить товар, которого уже нет в корзине, система не должна падать с ошибкой и должна корректно обновить отображение корзины.

## REQ_CART_003 — Изменение количества товара

3.1. Пользователь должен иметь возможность изменять количество товара в корзине.  
3.2. Количество товара не может быть меньше 1.  
3.3. Количество товара для одной позиции не может превышать 50; при попытке установить значение больше 50 система должна ограничить количество 50 и/или отобразить сообщение об ошибке.  
3.4. При изменении количества товара стоимость позиции должна пересчитываться автоматически.  
3.5. Изменения должны отображаться пользователю без перезагрузки страницы.  
3.6. При увеличении количества стоимость позиции увеличивается, при уменьшении — уменьшается.

## REQ_CART_004 — Расчёт итоговой стоимости корзины

4.1. Система должна рассчитывать итоговую стоимость товаров в корзине.  
4.2. Итоговая стоимость должна изменяться при:
- 4.2.1. Добавлении товара.
- 4.2.2. Удалении товара.
- 4.2.3. Изменении количества товара.

4.3. Итоговая стоимость должна отображаться пользователю в корзине.  
4.4. Стоимость каждого товара должна учитываться при расчёте общей суммы (цена товара × количество).  
4.5. Итоговая стоимость корзины равна сумме стоимостей всех позиций.  
4.6. Если в корзине нет товаров, отображается сообщение "Корзина пуста", итоговая стоимость не отображается.  
4.7. Стоимость товара и итоговая стоимость корзины должны отображаться с указанием валюты ("₽") и с двумя знаками после запятой.  
4.8. При изменении цен итоговая стоимость должна пересчитываться автоматически.  
4.9. Итоговая стоимость корзины не может превышать 100 000 000,00 рублей; при попытке добавить товар или изменить количество так, что общая сумма корзины станет больше 100 000 000,00 рублей:
- Операция не должна выполняться.
- Пользователю должно отображаться сообщение об ошибке ("Сумма корзины не может превышать 100 000 000").

## REQ_CART_005 — Просмотр корзины

5.1. Пользователь должен иметь возможность просматривать содержимое корзины.  
5.2. В корзине для каждого товара отображаются:
- 5.2.1. Наименование товара.
- 5.2.2. Цена товара.
- 5.2.3. Выбранное количество.

5.3. В корзине отображается итоговая стоимость всех товаров.  
5.4. Если в корзине нет товаров, отображается сообщение "Корзина пуста".
