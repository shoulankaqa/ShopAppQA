# REQ_AUTH_001 - Требования к регистрации

## Требования к логину

1. Логин должен содержать минимум 6 символов и максимум 32 символа.
2. Логин может состоять из:
   - латинских букв нижнего регистра [a-z],
   - цифр [0-9],
   - нижнего подчёркивания.
3. Логин должен начинаться с буквы.
4. Поле логина является обязательным.
5. Логин должен быть уникальным.
6. Запрещены логины, начинающиеся на `admin`, `user` или `seller`.

## Требования к паролю

1. Пароль должен содержать минимум 8 символов и максимум 64 символа.
2. Пароль должен содержать:
   - только латинские буквы (A-Z, a-z), цифры (0-9) и символы ($!^@?),
   - хотя бы одну заглавную букву (A-Z),
   - хотя бы одну цифру (0-9),
   - хотя бы один спецсимвол ($!^@?),
   - минимум 4 буквы всего.
3. Поле пароля является обязательным.
4. Поле пароля должно иметь иконку для переключения видимости пароля ("показать/скрыть пароль"), доступную с клавиатуры и с видимым фокусом.

## Процесс регистрации пользователя

1. Пользователь должен иметь возможность зарегистрироваться с использованием логина и пароля.
2. После успешной регистрации пользователь должен быть перенаправлен в раздел "Товары".
3. После успешной регистрации система должна отображать сообщение:  
   `"Успех. Зарегистрировались и авторизовались под: <имя пользователя>"`.
4. При попытке регистрации существующего пользователя система должна отображать сообщение:  
   `"Пользователь <имя пользователя> уже существует"`.
5. При наличии ошибок валидации (некорректный логин или пароль) система должна:
   - не создавать пользователя,
   - не авторизовывать пользователя,
   - отобразить сообщения об ошибках у соответствующих полей:
     - для поля логин: `"Логин должен содержать от 6 до 32 символов (a–z, 0–9, _), начинаться с буквы"`;
     - для поля пароль: `"Пароль должен содержать минимум 8 символов, включая хотя бы одну заглавную букву, одну цифру, один спецсимвол ($!^@?) и минимум 4 буквы"`.
6. Форма регистрации должна быть доступна для навигации с клавиатуры и иметь видимый фокус на активных элементах.
7. При регистрации новая учётная запись **должна создаваться с ролью обычного пользователя (базовая пользовательская роль по умолчанию)**; роли `admin` и `seller` не могут быть выбраны или назначены через форму саморегистрации.
