## REQ_ORDER_001 — Оформление заказа

1. Пользователь должен иметь возможность оформить заказ на товары, находящиеся в корзине.
2. Оформление заказа доступно только авторизованному пользователю.

### Запуск оформления заказа

3. Переход к оформлению заказа должен быть доступен из страницы "Корзина" по кнопке "Заказать".
4. Кнопка "Заказать" должна быть недоступна, если в корзине нет товаров.

### Данные для заказа

6. При оформлении заказа пользователь должен видеть:
   - список товаров из корзины (наименование, цена, количество, сумма по позиции),
   - итоговую стоимость заказа.
7. Для оформления заказа пользователь должен указать контактные данные, минимум:
   - имя получателя,
   - телефон,
   - адрес доставки (если доставка предусмотрена в проекте).
8. Все обязательные поля должны быть явно отмечены (например, звездочкой) и проверяться на заполнение.

### Валидация и ошибки

9. При отсутствии обязательных данных система не должна создавать заказ.
10. При ошибках валидации система должна:
    - не создавать заказ,
    - подсветить поля с ошибками,
    - отобразить понятные сообщения об ошибке (например: "Заполните имя", "Укажите телефон").

### Подтверждение заказа

11. После успешного оформления заказа система должна:
    - создать запись заказа в системе,
    - очистить корзину пользователя,
    - перенаправить пользователя на страницу подтверждения заказа.
12. На странице подтверждения система должна отображать:
    - номер заказа,
    - список заказанных товаров,
    - итоговую стоимость,
    - базовое сообщение, например: "Заказ успешно оформлен".

### Доступность и UX

13. Все элементы формы оформления заказа (поля ввода, кнопки) должны быть доступны для навигации с клавиатуры и иметь видимый фокус.
14. При обновлении суммы заказа (если предусмотрено изменение данных на шаге оформления) итоговая стоимость должна пересчитываться без перезагрузки страницы.
