# REQ_PRODUCT_001 — Просмотр товаров

1.1. Авторизованный пользователь должен иметь возможность просматривать список товаров.  

1.2. Для каждого товара в списке должны отображаться:
- 1.2.1. Название.
- 1.2.2. Цена.
- 1.2.3. Наличие товара.

1.3. Цена товара должна отображаться в формате `999 999,99`  
(с пробелами в качестве разделителей тысяч и запятой в качестве десятичного разделителя).

1.4. Если в системе нет ни одного товара, пользователю отображается сообщение `"Товаров пока нет"`.

1.5. Список товаров и элементы управления (фильтры, настройка количества) должны быть доступны для навигации с клавиатуры и иметь видимый фокус.

# REQ_PRODUCT_002 — Создание товара (admin/seller)

2.1. Пользователи с ролями `admin` и `seller` должны иметь возможность создавать новые товары.

2.2. При создании товара обязательными для заполнения являются поля:
- 2.2.1. "Наименование".
- 2.2.2. "Цена".

2.3. При создании товара доступен чекбокс наличия:
- 2.3.1. Если чекбокс установлен — товар считается "В наличии".
- 2.3.2. Если чекбокс не установлен — товар считается "Не в наличии".

2.4. Валидация поля "Цена":
- 2.4.1. Поле является обязательным.
- 2.4.2. Значение должно быть числом (целым или с копейками).
- 2.4.3. Минимальное значение: 1.
- 2.4.4. Максимальное значение: 1 000 000.
- 2.4.5. Отрицательные значения и 0 недопустимы.

2.5. Валидация поля "Наименование":
- 2.5.1. Поле является обязательным (минимум 1 символ).
- 2.5.2. Максимальная длина — 70 символов.
- 2.5.3. Может содержать буквы, цифры и спецсимволы.
- 2.5.4. В поле можно ввести не более 70 символов (71-й символ не вводится).

2.6. При успешном создании товар сохраняется и отображается в каталоге. Цена товара в каталоге должна отображаться в формате `999 999,99`.

2.7. При наличии ошибок валидации система должна:
- 2.7.1. Не создавать товар.
- 2.7.2. Подсветить поля с ошибками.
- 2.7.3. Отобразить сообщения об ошибке под соответствующими полями.

2.8. Доступ к созданию товаров должен быть ограничен только пользователями с ролями `admin` и `seller`.

# REQ_PRODUCT_003 — Редактирование товара (admin/seller)

3.1. Пользователи с ролями `admin` и `seller` должны иметь возможность редактировать существующие товары.

3.2. При редактировании товара изменения сохраняются только после явного подтверждения (нажатия кнопки "Сохранить").  
При отображении в каталоге цена товара должна быть показана в формате `999 999,99`.

3.3. При редактировании доступны следующие поля:
- 3.3.1. "Наименование" товара.
- 3.3.2. "Цена" товара.
- 3.3.3. Чекбокс наличия.

3.4. Поля "Наименование" и "Цена" являются обязательными для заполнения:
- 3.4.1. При пустом значении изменения не сохраняются, отображается сообщение об ошибке.

3.5. Для поля "Наименование" действуют ограничения:
- 3.5.1. Минимальная длина — 1 символ.
- 3.5.2. Максимальная длина — 70 символов.

3.6. Для поля "Цена" действуют ограничения:
- 3.6.1. Значение должно быть числом (целым или с копейками).
- 3.6.2. Минимальное значение: 1.
- 3.6.3. Максимальное значение: 1 000 000.
- 3.6.4. Отрицательные значения и 0 недопустимы.

3.7. При наличии ошибок валидации система должна:
- 3.7.1. Не сохранять изменения.
- 3.7.2. Подсветить поля с ошибками.
- 3.7.3. Отобразить сообщения об ошибке под соответствующими полями.

3.8. Доступ к редактированию товаров должен быть ограничен только пользователями с ролями `admin` и `seller`.

# REQ_PRODUCT_004 — Удаление товара (admin)

4.1. Пользователь с ролью `admin` должен иметь возможность удалить товар.

4.2. После удаления товар не должен отображаться в каталоге.

4.3. Система должна запрашивать подтверждение удаления через модальное окно.

4.4. После успешного удаления система должна отображать уведомление:  
`"Товар "<название>" удален!"`

4.5. Если при удалении произошла ошибка, система должна отобразить сообщение об ошибке и не изменять список товаров.

# REQ_PRODUCT_005 — Настройка количества отображаемых товаров

5.1. Пользователь должен иметь возможность выбрать количество товаров на странице.

5.2. Система должна предоставлять значения: 5, 10, 15, 20, 50, 100.

5.3. После изменения количества список должен обновляться автоматически.

5.4. Выбранное значение должно применяться к текущему списку товаров.

5.5. Выбранное значение не должно сохраняться после обновления страницы.

5.6. Настройка должна работать совместно с фильтрацией и сортировкой.

# REQ_PRODUCT_006 — Фильтрация товаров по доступности

6.1. Пользователь должен иметь возможность фильтровать товары по наличию на складе.

6.2. Система поддерживает состояния фильтра: "Все товары", "В наличии", "Нет в наличии".

6.3. При изменении фильтра список обновляется без перезагрузки страницы.

6.4. Фильтр должен работать совместно с другими настройками (например, количеством товаров на странице).

6.5. Если подходящих товаров нет, отображается сообщение `"Товаров пока нет"`.

6.6. Элементы управления фильтром должны быть доступны для навигации с клавиатуры и иметь видимый фокус.
