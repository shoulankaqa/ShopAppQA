## REQ_PRODUCT_001 — Просмотр товаров

1. Авторизованный пользователь должен иметь возможность просматривать список товаров.
2. Для каждого товара должны отображаться:
   - название,
   - цена,
   - наличие товара.
3. Если в системе нет ни одного товара, пользователю отображается сообщение "Товаров пока нет".
4. Список товаров и элементы управления (фильтры, настройка количества отображаемых товаров) должны быть доступны для навигации с клавиатуры и иметь видимый фокус.

## REQ_PRODUCT_002 — Создание товара (admin/seller)

1. Пользователи с ролями "admin" и "seller" должны иметь возможность создавать новые товары в разделе "Товары".
2. При создании товара обязательными для заполнения являются поля:
   - "Наименование",
   - "Цена".
3. При создании товара доступен чекбокс наличия:
   - если чекбокс установлен — товар считается "В наличии" и доступен для покупки;
   - если чекбокс не установлен — товар считается "Не в наличии" и недоступен для покупки.
4. Поле "Цена":
   - является обязательным для заполнения;
   - значение должно быть числом (целым или с копейками);
   - минимальное значение: 1;
   - максимальное значение: 1 000 000;
   - отрицательные значения и 0 недопустимы.
5. Поле "Наименование":
   - является обязательным для заполнения (минимум 1 символ);
   - максимальная длина — 70 символов;
   - может содержать буквы, цифры и спецсимволы;
   - в поле можно ввести не более 70 символов (71‑й символ не вводится).
6. При успешном создании товар сохраняется в системе и отображается в каталоге товаров.
7. При наличии ошибок валидации (некорректное "Наименование" или "Цена") система должна:
   - не создавать товар,
   - подсветить поля с ошибками,
   - отобразить сообщения об ошибке под соответствующими полями.
8. Доступ к созданию товаров должен быть ограничен только пользователями с ролями "admin" и "seller"; остальные пользователи не должны видеть элементы управления для создания товара.

## REQ_PRODUCT_003 — Редактированиe товара (admin/seller)

1. Пользователи с ролями "admin" и "seller" должны иметь возможность редактировать существующие товары в разделе "Товары".
2. При редактировании товара изменения сохраняются только после явного подтверждения (нажатия кнопки "Сохранить").
3. При редактировании товара доступны следующие поля:
   - "Наименование" товара,
   - "Цена" товара,
   - чекбокс наличия:
     - "В наличии" — товар доступен для покупки;
     - "Не в наличии" — товар недоступен для покупки.
4. Поля "Наименование" и "Цена" являются обязательными для заполнения:
   - при пустом значении изменения не сохраняются,
   - пользователю отображается сообщение об ошибке под соответствующим полем.
5. Для поля "Наименование" действуют следующие ограничения:
   - минимальная длина — 1 символ;
   - максимальная длина — 70 символов;
   - может содержать буквы, цифры и спецсимволы;
   - в поле можно ввести не более 70 символов (71‑й символ не вводится).
6. Для поля "Цена" действуют следующие ограничения:
   - значение должно быть числом (целым или с копейками);
   - минимальное значение: 1;
   - максимальное значение: 1 000 000;
   - отрицательные значения и 0 недопустимы;
   - при некорректном значении (0, отрицательное, не число, более 1 000 000) изменения не сохраняются, пользователю отображается сообщение об ошибке.
7. При наличии ошибок валидации система должна:
   - не сохранять изменения,
   - подсветить поля с ошибками,
   - отобразить сообщения об ошибке под соответствующими полями.
8. Доступ к редактированию товаров должен быть ограничен только пользователями с ролями "admin" и "seller".

## REQ_PRODUCT_004 — Удаление товара (admin)

1. Пользователь с ролью "admin" должен иметь возможность удалить товар.
2. После удаления товар не должен отображаться в каталоге.
3. Система должна запрашивать подтверждение удаления через модальное окно.
4. После успешного удаления система должна отображать уведомление:
   "Товар "<название>" удален!".
5. Если при удалении товара произошла ошибка, система должна отобразить сообщение об ошибке и не изменять список товаров.

## REQ_PRODUCT_005 — Настройка количества отображаемых товаров

1. Пользователь должен иметь возможность выбрать количество товаров, отображаемых на странице.
2. Система должна предоставлять следующие значения для выбора:
   - 5,
   - 10,
   - 15,
   - 20,
   - 50,
   - 100.
3. После изменения количества отображаемых товаров список должен обновляться автоматически.
4. Выбранное значение должно применяться к текущему списку товаров.
5. Выбранное значение не должно сохраняться после обновления страницы.
6. Настройка количества отображаемых товаров должна работать совместно с фильтрацией и сортировкой (если сортировка предусмотрена).

## REQ_PRODUCT_006 — Фильтрация товаров по доступности

1. Пользователь должен иметь возможность фильтровать товары по наличию на складе.
2. Система поддерживает 3 состояния фильтра:
   - "Все товары":
     > отображаются все товары без фильтрации;
   - "В наличии":
     > отображаются товары, которые есть в наличии;
   - "Нет в наличии":
     > отображаются товары, которых нет в наличии.
3. При изменении фильтра список товаров обновляется без перезагрузки страницы.
4. Фильтр применяется ко всем товарам каталога и должен работать совместно с другими настройками (например, количеством товаров на странице).
5. Если подходящих товаров нет, отображается сообщение "Товаров пока нет".
6. Элементы управления фильтром должны быть доступны для навигации с клавиатуры и иметь видимый фокус.
