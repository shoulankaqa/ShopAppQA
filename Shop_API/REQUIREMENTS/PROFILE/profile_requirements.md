## REQ_PROFILE_001 — Общие требования к профилю

1.1. Пользователь должен иметь возможность просматривать и изменять данные профиля после авторизации.  
1.2. Изменения в профиле сохраняются только после явного подтверждения (нажатия кнопки "Сохранить").  
1.3. После успешного сохранения изменений система должна отобразить пользователю сообщение об успешном обновлении профиля.  
1.4. При ошибке сохранения (проблемы на сервере, недоступность сервиса) изменения не применяются, пользователю отображается сообщение об ошибке.  
1.5. Поля профиля, доступные для редактирования, должны проходить валидацию на стороне клиента и сервера (при наличии соответствующих правил).

## REQ_PROFILE_002 — Управление email

2.1. Пользователь должен иметь возможность добавить или изменить email в профиле.  
2.2. Email используется для восстановления пароля и рассылки сервисных уведомлений.  
2.3. Введённый email должен соответствовать формату валидного адреса электронной почты (наличие символа "@", домена, отсутствие недопустимых символов).  
2.4. При вводе некорректного email система должна:
- 2.4.1. Не сохранять изменения.
- 2.4.2. Подсветить поле email.
- 2.4.3. Отобразить сообщение об ошибке под полем.

2.5. При изменении email система должна сохранять новое значение только после успешной валидации.

## REQ_PROFILE_003 — Изменение пароля

3.1. Пользователь должен иметь возможность изменить пароль после авторизации.  
3.2. Для изменения пароля необходимо:
- 3.2.1. Ввести текущий пароль.
- 3.2.2. Ввести новый пароль.
- 3.2.3. Повторно ввести новый пароль для подтверждения.

3.3. Новый пароль должен соответствовать требованиям REQ_AUTH_001 — Требования к регистрации (раздел "Требования к паролю").  
3.4. Если текущий пароль введён неверно, система:
- 3.4.1. Не изменяет пароль.
- 3.4.2. Отображает сообщение об ошибке о неверном текущем пароле.

3.5. Если новый пароль не соответствует требованиям безопасности или не совпадает с подтверждением (при наличии поля подтверждения), система:
- 3.5.1. Не изменяет пароль.
- 3.5.2. Подсвечивает соответствующие поля.
- 3.5.3. Отображает сообщения об ошибке под полями.

3.6. После успешной смены пароля система должна:
- 3.6.1. Сохранить новый пароль.
- 3.6.2. Отобразить сообщение об успешной смене пароля.
