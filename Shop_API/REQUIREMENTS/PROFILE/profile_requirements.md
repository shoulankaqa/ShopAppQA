## REQ_PROFILE_001 — Общие требования к профилю

1. Пользователь должен иметь возможность изменить данные профиля после авторизации.
2. Изменения должны сохраняться после подтверждения.

---

## REQ_PROFILE_002 — Управление email

1. Пользователь может добавить или изменить email в профиле.
2. Email используется для восстановления пароля.
3. Email должен соответствовать формату example@domain.com.
4. При изменении email система сохраняет новое значение.

 ---

## REQ_PROFILE_003 — Изменениe пароля

1. Пользователь должен иметь возможность изменить пароль.
2. Для изменения пароля необходимо ввести текущий пароль.
3. Новый пароль должен соответствовать требованиям REQ_AUTH_002 — Требования к паролю.

## REQ_PROFILE_001 — Общие требования к профилю

1. Пользователь должен иметь возможность просматривать и изменять данные профиля после авторизации.
2. Изменения в профиле сохраняются только после явного подтверждения (нажатия кнопки "Сохранить").
3. После успешного сохранения изменений система должна отобразить пользователю сообщение об успешном обновлении профиля.
4. При ошибке сохранения (проблемы на сервере, недоступность сервиса) изменения не применяются, пользователю отображается сообщение об ошибке.
5. Поля профиля, доступные для редактирования, должны проходить валидацию на стороне клиента и сервера (при наличии соответствующих правил).

---

## REQ_PROFILE_002 — Управление email

1. Пользователь должен иметь возможность добавить или изменить email в профиле.
2. Email используется для восстановления пароля и рассылки сервисных уведомлений.
3. Введённый email должен соответствовать формату валидного адреса электронной почты (наличие символа "@" и домена, отсутствие недопустимых символов).
4. При вводе некорректного email система должна:
   - не сохранять изменения,
   - подсветить поле email,
   - отобразить сообщение об ошибке под полем.
5. При изменении email система должна сохранять новое значение только после успешной валидации.
---

## REQ_PROFILE_003 — Изменениe пароля

1. Пользователь должен иметь возможность изменить пароль после авторизации.
2. Для изменения пароля необходимо:
   - ввести текущий пароль,
   - ввести новый пароль,
   - (опционально) повторно ввести новый пароль для подтверждения.
3. Новый пароль должен соответствовать требованиям REQ_AUTH_002 — Требования к паролю.
4. Если текущий пароль введён неверно, система:
   - не изменяет пароль,
   - отображает сообщение об ошибке о неверном текущем пароле.
5. Если новый пароль не соответствует требованиям безопасности или не совпадает с подтверждением (при наличии поля подтверждения), система:
   - не изменяет пароль,
   - подсвечивает соответствующие поля,
   - отображает сообщения об ошибке под полями.
6. После успешной смены пароля система должна:
   - сохранить новый пароль,
   - отобразить сообщение об успешной смене пароля.
