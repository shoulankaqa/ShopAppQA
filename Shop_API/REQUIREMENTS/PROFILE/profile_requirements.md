## REQ_PROFILE_001 — Просмотр профиля

1.1. Пользователь должен иметь возможность просматривать профиль.

1.2. В профиле должны отображаться данные пользователя в блоке "Информация об аккаунте".

1.3. В блоке "Информация об аккаунте" содержится:
 - имя пользователя, которое нельзя изменять;
 - электронная почта пользователя, которую можно изменять;
 - кнопка "Изменить" для редактирования электронной почты.

1.4. В блоке "Безопасность" содержится:
 - зашифрованный пароль пользователя, который можно изменить;
 - кнопка "Изменить" для редактирования пароля.

1.5. Профиль должен быть доступен только авторизованному пользователю.


## REQ_PROFILE_002 — Управление email

2.1. Пользователь должен иметь возможность добавить или изменить email в профиле в блоке "Информация об аккаунте".

2.2. Форма редактирования электронной почты открывается по нажатию кнопки "Изменить" в блоке "Информация об аккаунте".

2.3. Email используется для восстановления пароля.

2.4. Введённый email должен соответствовать формату валидного адреса электронной почты (наличие символа "@", домена, отсутствие недопустимых символов).

2.5. При вводе некорректного email система:
- не сохраняет изменения;
- отображает сообщение об ошибке под полем.

2.6. При изменении email система должна сохранять новое значение только после успешной валидации и явного подтверждения (нажатия кнопки "Сохранить" в блоке редактирования электронной почты).

2.7. При нажатии кнопки "Отмена" в блоке редактирования электронной почты:
- изменения не сохраняются;
- значения поля возвращаются к состоянию до начала редактирования;
- форма редактирования электронной почты закрывается.

2.8. Если электронная почта уже добавлена в профиль, пользователь не может удалить её полностью: при сохранении изменений поле "Электронная почта" не может быть пустым.

2.9. Если введённый адрес электронной почты не содержит символа "@", система:
- не сохраняет изменения;
- отображает сообщение об ошибке:  
  `Адрес электронной почты должен содержать символ @. В адресе "<адрес>" отсутствует символ @.`

2.10. Если после символа "@" отсутствует доменная часть (нет символов после "@"), система:
- не сохраняет изменения;
- отображает сообщение об ошибке:  
  `Введите часть адреса после символа @. Адрес "<адрес>" неполный.`

2.11. Если в доменной части после символа "@" точка стоит в недопустимом положении (точка сразу после "@", либо точка является последним символом, и после неё нет ни одного символа), система:
- не сохраняет изменения;
- отображает сообщение об ошибке:  
  `Недопустимое положение символа "." в адресе "<доменная часть после @>".`

2.12. Если в локальной части адреса (до символа "@") содержится русская буква, система:
- не сохраняет изменения;
- отображает сообщение об ошибке:  
  `Часть адреса до символа @ не должна содержать символ "<русская буква>".`


## REQ_PROFILE_003 — Изменение пароля

3.1. Пользователь должен иметь возможность изменить пароль после авторизации в блоке "Безопасность" через кнопку "Изменить пароль".

3.2. Для изменения пароля необходимо:
- ввести текущий пароль;
- ввести новый пароль;
- повторно ввести новый пароль для подтверждения.

3.3. Новый пароль должен соответствовать требованиям REQ_AUTH_001 — Требования к регистрации (раздел "Требования к паролю").

3.4. Если текущий пароль введён неверно, система:
- не изменяет пароль;
- отображает сообщение об ошибке: `Request failed with status code 400`.

3.5. Если новый пароль и его подтверждение не совпадают, система:
- не изменяет пароль;
- отображает сообщение об ошибке: `Пароли не совпадают`.

3.6. Если длина нового пароля меньше минимально допустимого значения (8 символов), система:
- не изменяет пароль;
- отображает сообщение об ошибке под полем нового пароля: `Минимально допустимое кол-во символов 8. Длина текста сейчас: <количество текущих символов>`.

3.7. При попытке сохранить изменения пароля при незаполненном одном или нескольких обязательных полях
("Текущий пароль", "Новый пароль", "Подтверждение нового пароля") система:
- не изменяет пароль;
- отображает под каждым незаполненным полем сообщение об ошибке: `Заполните это поле` и визуально подсвечивает поле как ошибочное.

3.8. Если значения в полях нового пароля и подтверждения пароля (при совпадении друг с другом) не соответствуют требованиям безопасности, система:
- не изменяет пароль;
- отображает сообщение об ошибке: `Request failed with status code 400`.

3.9. В модуле изменения пароля должна быть доступна кнопка "Отмена", при нажатии которой:
- изменения пароля не применяются;
- форма изменения пароля закрывается (или возвращается к исходному состоянию без изменений).

3.10. После успешной смены пароля система должна:
- сохранить новый пароль;
- отобразить сообщение об успешной смене пароля.

3.11. Для полей ввода пароля ("Текущий пароль", "Новый пароль", "Подтверждение нового пароля") должна быть доступна функция "показать/скрыть" пароль с помощью иконки-глаз:
- по умолчанию значение в поле отображается скрытым;
- при нажатии на иконку глаза пароль отображается открыто;
- повторное нажатие на иконку возвращает поле в скрытый режим.


## REQ_PROFILE_004 — Выход из аккаунта

4.1. На странице "Профиль" в блоке "Выход из аккаунта" должна отображаться кнопка "Выйти".

4.2. При нажатии кнопки "Выйти" пользователь должен быть разлогинен в системе на текущем устройстве.

4.3. После выхода из аккаунта пользователь должен быть перенаправлен на страницу авторизации.

4.4. После выхода данные профиля больше не должны отображаться до следующей авторизации.
