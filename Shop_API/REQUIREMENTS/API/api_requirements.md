## REQ_API_001 — Формат запросов

1. API должно принимать запросы в формате JSON.

---

## REQ_API_002 — Формат оветов

1. API должно возвращать ответы в формате JSON.

---

## REQ_API_003 — HTTP status codes

1. API должно использовать корректные HTTP status codes:
 - 200 OK
 - 201 Created
 - 204 No Content
 - 400 Bad Request
 - 401 Unauthorized
 - 403 Forbidden
 - 404 Not Found
 - 409 Conflict
 - 500 Internal Server Error.

 ---

 ## REQ_API_004 — JWT авторизация

 1. API должно использовать JWT token для авторизации.

---
