## API_CART_001 — Получение корзины текущего пользователя

1. Назначение
   - Метод предназначен для получения содержимого корзины текущего авторизованного пользователя.

2. URL и HTTP‑метод
   - HTTP‑метод: GET
   - URL: /cart

3. Доступ и авторизация
   - Метод доступен только авторизованным пользователям (ролям, которым разрешена работа с корзиной — `user`, `seller`, `admin` согласно общим требованиям).
   - Запрос должен выполняться с валидным `access_token` в заголовке `Authorization: Bearer <access_token>`.

4. Тело запроса
   - Отсутствует.
   - Дополнительные параметры не используются.

5. Ответ при успехе

   - Код ответа: `200 OK`.
   - Формат: `application/json`.

   Структура ответа:

```json
{
  "id": 0,
  "items": [
    {
      "id": 0,
      "product": {
        "id": 0,
        "name": "string",
        "price": 1,
        "available": true
      },
      "quantity": 0
    }
  ]
}
```

   Описание полей:
   - `id` (integer) — идентификатор корзины текущего пользователя.
   - `items` (array) — список позиций в корзине.
     - `items[].id` (integer) — идентификатор позиции корзины.
     - `items[].product` — информация о товаре:
       - `id` (integer) — идентификатор товара.
       - `name` (string) — наименование товара.
       - `price` (number) — цена товара.
       - `available` (boolean) — признак того, доступен ли товар к покупке.
     - `items[].quantity` (integer) — количество единиц товара в корзине.

   Правила:
   - Если в корзине нет ни одной позиции, должен возвращаться объект корзины с пустым массивом `items` (а не ошибка).

6. Ошибки

   - `401 Unauthorized` — пользователь не авторизован или `access_token` отсутствует/некорректен.

```json
{
  "code": 401,
  "message": "string"
}
```

   - `403 Forbidden` — у текущей роли недостаточно прав для работы с корзиной.

```json
{
  "code": 403,
  "message": "string"
}
```

   - `500 Internal Server Error` — внутренняя ошибка сервера.

7. Особые условия

   - Метод возвращает корзину именно **текущего** пользователя, определённого по `access_token`; передавать id пользователя или корзины в запросе нельзя.


## API_CART_002 — Добавление товара в корзину или изменение его количества

1. Назначение
   - Метод предназначен для добавления товара в корзину текущего пользователя или изменения количества уже добавленного товара.

2. URL и HTTP‑метод
   - HTTP‑метод: POST
   - URL: /cart/items

3. Доступ и авторизация
   - Метод доступен только авторизованным пользователям (ролям, которым разрешена работа с корзиной — `user`, `seller`, `admin` согласно общим требованиям).
   - Запрос должен выполняться с валидным `access_token` в заголовке `Authorization: Bearer <access_token>`.

4. Тело запроса

   Формат: `application/json`.

   Обязательные поля:
   - `productId` (integer) — идентификатор товара, который добавляется/изменяется в корзине.
   - `quantity` (integer) — требуемое количество товара в корзине.

   Правила:
   - `productId` и `quantity` являются обязательными; отсутствие любого из них приводит к ошибке `400 Bad Request`.
   - При `quantity > 0`:
     - если товара ещё нет в корзине — он добавляется с указанным количеством;
     - если товар уже есть в корзине — его количество обновляется до указанного значения (или увеличивается/уменьшается, в зависимости от принятой бизнес‑логики; это нужно явно указать в общих требованиях к корзине).
   - При `quantity <= 0`:
     - поведение нужно зафиксировать отдельно:
       - либо считать запрос некорректным (`400 Bad Request`);
       - либо интерпретировать как удаление позиции из корзины.
   - Товар может быть добавлен/изменён только если он существует и доступен для покупки (например, `available = true` в модели товара); при недоступном товаре возвращается `404`.

   Пример запроса:

```http
POST /cart/items HTTP/1.1
Host: api.example.com
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "productId": 42,
  "quantity": 3
}
```

5. Ответ при успехе

   - Код ответа: `200 OK`.
   - Формат: `application/json`.

   Структура ответа:

```json
{
  "id": 0,
  "product": {
    "id": 0,
    "name": "string",
    "price": 0,
    "available": true
  },
  "quantity": 0
}
```

   Описание полей:
   - `id` (integer) — идентификатор позиции корзины.
   - `product` — сведения о товаре:
     - `id` (integer) — идентификатор товара;
     - `name` (string) — наименование товара;
     - `price` (number) — цена товара;
     - `available` (boolean) — признак доступности товара.
   - `quantity` (integer) — фактическое количество товара в корзине после обработки запроса.

6. Ошибки

   - `400 Bad Request` — некорректный запрос (обязательные поля не переданы, неверные значения).
     - Возможные причины:
       - отсутствует `productId` или `quantity`;
       - `quantity` не соответствует допустимым значениям (например, меньше 1, если так заведено в требованиях).
     - Формат ответа:

```json
{
  "code": 400,
  "message": "string"
}
```

   - `401 Unauthorized` — пользователь не авторизован или `access_token` отсутствует/некорректен.

```json
{
  "code": 401,
  "message": "string"
}
```

   - `403 Forbidden` — у текущей роли недостаточно прав для работы с корзиной.

```json
{
  "code": 403,
  "message": "string"
}
```

   - `404 Not Found` — товар не найден или недоступен (например, `available = false`).

```json
{
  "code": 404,
  "message": "string"
}
```

   - `500 Internal Server Error` — внутренняя ошибка сервера (если используется).

7. Особые условия

   - Метод работает только с корзиной **текущего** пользователя, определённого по `access_token` — `productId` относится к продукту, а не к чужой корзине.
   - Логика “создать или обновить” должна быть согласована с функциональными требованиями к корзине (REQ_CART_001: добавление/изменение количества при повторных вызовах).


## API_CART_003 — Изменение количества товара в корзине

1. Назначение
   - Метод предназначен для изменения количества товара в корзине текущего пользователя по идентификатору позиции корзины.
   - При количестве `quantity > 0` позиция обновляется, при `quantity == 0` — удаляется.

2. URL и HTTP‑метод
   - HTTP‑метод: PATCH
   - URL: /cart/items/{id}

3. Доступ и авторизация
   - Метод доступен только авторизованным пользователям (ролям, которым разрешена работа с корзиной - `user`, `seller`, `admin`).
   - Запрос должен выполняться с валидным `access_token` в заголовке `Authorization: Bearer <access_token>`.

4. Параметры запроса

   Path‑параметры:
   - `id` (string, uuid, required) — идентификатор позиции корзины, которую нужно изменить или удалить.

5. Тело запроса

   Формат: `application/json`.

   Обязательные поля:
   - `quantity` (integer, required, minimum: 0) — новое количество товара в корзине для данной позиции.

   Правила:
   - `quantity` обязателен и должен быть `>= 0`.
   - Если `quantity > 0`:
     - позиция корзины обновляется с указанным количеством;
     - в ответе возвращается обновлённая позиция (код `200 OK`).
   - Если `quantity == 0`:
     - позиция корзины удаляется;
     - возвращается код `204 No Content`, тело отсутствует.
   - Значение `quantity < 0` считается некорректным и приводит к ошибке `400 Bad Request`.

   Пример запроса (изменение количества):

```http
PATCH /cart/items/04167fbf-c7a7-433a-997e-6394ae51ebb5 HTTP/1.1
Host: api.example.com
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "quantity": 3
}
```

   Пример запроса (удаление позиции):

```http
PATCH /cart/items/04167fbf-c7a7-433a-997e-6394ae51ebb5 HTTP/1.1
Host: api.example.com
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "quantity": 0
}
```

6. Ответ при успехе

   - Вариант 1: позиция обновлена (`quantity > 0`):
     - Код ответа: `200 OK`.
     - Формат: `application/json`.

```json
{
  "id": 0,
  "product": {
    "id": 0,
    "name": "string",
    "price": 0,
    "available": true
  },
  "quantity": 0
}
```

   - Вариант 2: позиция удалена (`quantity == 0`):
     - Код ответа: `204 No Content`.
     - Тело ответа отсутствует.

7. Ошибки

   - `400 Bad Request` — некорректный запрос.
     - Причины:
       - отсутствует поле `quantity`;
       - `quantity < 0`;
       - некорректный формат тела запроса.
     - Формат:

```json
{
  "code": 400,
  "message": "string"
}
```

   - `401 Unauthorized` — пользователь не авторизован или `access_token` отсутствует/некорректен.

```json
{
  "code": 401,
  "message": "string"
}
```

   - `403 Forbidden` — у текущей роли недостаточно прав для работы с корзиной.

```json
{
  "code": 403,
  "message": "string"
}
```

   - `404 Not Found` — позиция корзины с указанным `id` не найдена (либо принадлежит другому пользователю).

```json
{
  "code": 404,
  "message": "string"
}
```

8. Особые условия

   - Метод работает только с позициями корзины **текущего** пользователя, определённого по `access_token`; изменение или удаление позиции другой корзины недопустимо и должно приводить к `404 Not Found` (без раскрытия информации о чужих корзинах).
   - Логика удаления при `quantity == 0` должна быть согласована с функциональными требованиями к корзине (REQ_CART_001 и др.).


## API_CART_004 — Удаление позиции из корзины

1. Назначение
   - Метод предназначен для удаления указанной позиции из корзины текущего авторизованного пользователя.

2. URL и HTTP‑метод
   - HTTP‑метод: DELETE
   - URL: /cart/items/{id}

3. Доступ и авторизация
   - Метод доступен только авторизованным пользователям (ролям, которым разрешена работа с корзиной - `user`, `seller`, `admin`).
   - Запрос должен выполняться с валидным `access_token` в заголовке `Authorization: Bearer <access_token>`.

4. Параметры запроса

   Path‑параметры:
   - `id` (string, uuid, required) — идентификатор позиции корзины, которая должна быть удалена.

   Пример запроса:

```http
DELETE /cart/items/04167fbf-c7a7-433a-997e-6394ae51ebb5 HTTP/1.1
Host: api.example.com
Authorization: Bearer <access_token>
```

5. Ответ при успехе

   - Код ответа: `204 No Content`.
   - Тело ответа отсутствует.
   - Позиция корзины с указанным `id` должна быть удалена и больше не возвращаться в `GET /cart`.

6. Ошибки

   - `400 Bad Request` — неверные данные.
     - Возможные причины:
       - некорректный формат `id` (не uuid и т.п.).
     - Формат ответа:

```json
{
  "code": 400,
  "message": "string"
}
```

   - `401 Unauthorized` — пользователь не авторизован или `access_token` отсутствует/некорректен.

```json
{
  "code": 401,
  "message": "string"
}
```

   - `403 Forbidden` — у текущей роли недостаточно прав для работы с корзиной.

```json
{
  "code": 403,
  "message": "string"
}
```

   - `404 Not Found` — позиция корзины не найдена (либо не принадлежит текущему пользователю).

```json
{
  "code": 404,
  "message": "string"
}
```

7. Особые условия

   - Метод должен удалять только позиции корзины **текущего** пользователя, определённого по `access_token`; попытка удалить позицию чужой корзины должна приводить к `404 Not Found`, без раскрытия информации о существовании такой позиции.
   - Операция должна быть идемпотентной: повторный DELETE одной и той же позиции после успешного удаления может возвращать `404 Not Found` (позиция уже отсутствует).


## API_CART_005 — Оплата корзины из кошелька

1. Назначение
   - Метод предназначен для оплаты содержимого корзины текущего авторизованного пользователя за счёт средств кошелька: рассчитывает итоговую сумму, проверяет баланс, при достаточном количестве средств списывает сумму и очищает корзину.

2. URL и HTTP‑метод
   - HTTP‑метод: POST
   - URL: /v2/cart/checkout

3. Доступ и авторизация
   - Метод доступен только авторизованным пользователям, которым разрешена работа с корзиной и кошельком (`user`, `seller`, `admin`).
   - Запрос должен выполняться с валидным `access_token` в заголовке `Authorization: Bearer <access_token>`.

4. Тело запроса
   - Тело запроса отсутствует.
   - Все данные берутся из контекста текущего пользователя (корзина и кошелёк).

5. Логика работы

   При вызове метода система должна:

   1. Определить текущего пользователя по `access_token`.
   2. Получить его корзину и рассчитать итоговую сумму к оплате (с учётом актуальных цен товаров, их доступности и количества).
   3. Проверить, что корзина не пуста:
      - если корзина пуста — вернуть ошибку `400` с сообщением `"Корзина пуста"`;
   4. Получить текущий баланс кошелька пользователя.
   5. Сравнить баланс с суммой корзины:
      - если средств недостаточно — вернуть ошибку `409` с указанием текущего баланса и требуемой суммы;
      - если средств достаточно:
        - списать сумму корзины с баланса кошелька;
        - очистить корзину (удалить все позиции).
   6. Вернуть успешный ответ с информацией об операции и новом балансе.

6. Ответ при успехе

   - Код ответа: `200 OK`.
   - Формат: `application/json`.

```json
{
  "code": 200,
  "message": "Вы оплатили корзину на 1500 рублей",
  "newBalance": 8500
}
```

   Описание полей:
   - `code` (number) — код результата (200).
   - `message` (string) — человекочитаемое сообщение о результате оплаты с указанием суммы.
   - `newBalance` (number) — новый баланс кошелька пользователя после списания.

7. Ошибки

   - `400 Bad Request` — корзина пуста.

```json
{
  "code": 400,
  "message": "Корзина пуста"
}
```

   - `401 Unauthorized` — пользователь не авторизован или `access_token` отсутствует/некорректен.

```json
{
  "code": 401,
  "message": "string"
}
```

   - `403 Forbidden` — недостаточно прав для выполнения операции (роль пользователя не имеет права оплачивать корзину из кошелька).

```json
{
  "code": 403,
  "message": "string"
}
```

   - `409 Conflict` — недостаточно средств на балансе кошелька.

```json
{
  "code": 409,
  "message": "Недостаточно средств на балансе",
  "currentBalance": 500,
  "requiredAmount": 1500
}
```

   Описание дополнительных полей:
   - `currentBalance` (number) — текущий баланс кошелька пользователя.
   - `requiredAmount` (number) — сумма, необходимая для оплаты корзины.

   - (Опционально) `500 Internal Server Error` — внутренняя ошибка сервера.

8. Особые условия

   - Метод работает только с корзиной и кошельком **текущего** пользователя, определённого по `access_token`.
   - Операция должна быть атомарной: списание средств и очистка корзины должны выполняться как единая транзакция, чтобы исключить ситуации, когда деньги списаны, а корзина не очищена, или наоборот.
   - При повторном вызове метода сразу после успешной оплаты (когда корзина уже очищена) должна возвращаться ошибка `400` `"Корзина пуста"`.


## API_CART_006 — Очистка корзины текущего пользователя

1. Назначение
   - Метод предназначен для полной очистки корзины текущего авторизованного пользователя (удаление всех позиций из корзины).

2. URL и HTTP‑метод
   - HTTP‑метод: DELETE
   - URL:/cart/clear

   - Метод доступен только авторизованным пользователям (`user`, `seller`, `admin`).
   - Запрос должен выполняться с валидным `access_token` в заголовке `Authorization: Bearer <access_token>`.

        - удалить все позиции из корзины;

```json
{
  "code": 400,
```

