## API_USERS_001 — Получение списка пользователей (ADMIN, SELLER)


1. Назначение
   - Метод предназначен для получения списка всех пользователей системы.
   - Возвращает идентификаторы, логины (username) и роли пользователей.
   - Доступно только пользователям с ролями `admin` и `seller`.


2. URL и HTTP‑метод
   - HTTP‑метод: GET
   - URL: /users


3. Доступ и авторизация
   - Метод доступен только авторизованным пользователям с ролями `admin` и `seller`.
   - Запрос должен выполняться с валидным `access_token` в заголовке `Authorization: Bearer <access_token>`.


4. Параметры запроса
   - Параметры запроса (query/path/header) не требуются.


   Пример запроса:


```http
GET /users HTTP/1.1
Host: api.example.com
Authorization: Bearer <access_token>
```


5. Тело запроса
   - Тело запроса не требуется .

6. Ответ при успехе

   - Код ответа: `200 OK`.
   - Формат: `application/json`.
   - Тип: массив объектов пользователя.


```json
[
  {
    "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "username": "admin",
    "role": "user"
  }
]
```

   Описание полей объекта пользователя:
   - `id` (string) — уникальный идентификатор пользователя (UUID).
   - `username` (string) — логин пользователя.
   - `role` (string) — роль пользователя в системе (например, `user`, `admin`, `seller`).

7. Ошибки

   - `401 Unauthorized` — отсутствует или некорректный `access_token`.

```json
{
  "code": 401,
  "message": "string"
}
```
   Возможные причины:
   - отсутствует заголовок `Authorization`;
   - `access_token` недействителен, истёк или некорректно сформирован.


   - `403 Forbidden` — пользователь авторизован, но его роль не `admin` и не `seller` (недостаточно прав).

```json
{
  "message": "string"
}
```

   Возможные причины:
   - роль пользователя — `user` или другая, не разрешённая для этого метода.

8. Особые условия:
   - Метод возвращает список всех пользователей системы (без фильтрации по роли, статусу и т.д.).
   - В список включаются все пользователи, включая `admin`, `seller` и `user`.


## API_USERS_002 — Запрос на сброс пароля (FORGOT_PASSWORD)

1. Назначение
   - Метод предназначен для инициирования процесса сброса пароля пользователя.
   - При успешном запросе на email пользователя отправляется письмо с инструкциями по восстановлению пароля.
   - Доступно для всех пользователей (не требуется авторизация).

2. URL и HTTP‑метод
   - HTTP‑метод: POST
   - URL: /users/forgot-password

3. Доступ и авторизация
   - Метод доступен неавторизованным пользователям (не требуется `access_token`).
   - Запрос выполняется без заголовка `Authorization`.

4. Параметры запроса
   - Параметры запроса (query/path/header) не требуются.

5. Тело запроса
   - Формат: `application/json`.
   - Тип: строка, содержащая email пользователя.

```json
"user@example.com"
```

   Описание:
   - Строка должна содержать корректный email пользователя, зарегистрированного в системе.


   Пример запроса:


```http
POST /users/forgot-password HTTP/1.1
Host: api.example.com
Content-Type: application/json

"user@example.com"
```

6. Ответ при успехе


   - Код ответа: `200 OK`.
   - Формат: `application/json`.

```json
{
  "message": "string"
}
```

   Описание:
   - `message` (string) — подтверждение, что письмо с инструкциями отправлено на email (например, `"Письмо с инструкциями по восстановлению пароля отправлено на ваш email"`).


7. Ошибки

   - `400 Bad Request` — неверные данные (email не корректен или не передан).

```json
{
  "message": "string"
}
```

   Возможные причины:
   - тело запроса не является строкой;
   - строка не содержит корректный email (не соответствует формату email);
   - тело запроса пустое.


   - `404 Not Found` — пользователь с указанным email не найден в системе.


```json
{
  "message": "string"
}
```

   Возможные причины:
   - email не зарегистрирован в системе;
   - пользователь был удалён.


   - `500 Internal Server Error` — ошибка сервера при обработке запроса (например, ошибка при отправке письма).


```json
{
  "message": "string"
}
```

   Возможные причины:
   - ошибка почтового сервиса;
   - внутренняя ошибка сервера.


## API_USERS_003 — Установка нового пароля (RESET_PASSWORD)

1. Назначение
   - Метод предназначен для установки нового пароля пользователем после запроса на сброс пароля.
   - Для подтверждения используется уникальный токен, полученный в письме на email.
   - Доступно для всех пользователей (не требуется авторизация).

2. URL и HTTP‑метод
   - HTTP‑метод: POST
   - URL: /users/reset-password

3. Доступ и авторизация
   - Метод доступен неавторизованным пользователям (не требуется `access_token`).
   - Запрос выполняется без заголовка `Authorization`.

4. Параметры запроса
   - Параметры запроса (query/path/header) не требуются.

5. Тело запроса
   - Формат: `application/json`.
   - Тип: строка, содержащая токен для восстановления пароля.

```json
"token_string_here"
```

   Описание:
   - Строка должна содержать корректный токен, полученный в письме на email после запроса `/users/forgot-password`.


   Пример запроса:

```http
POST /users/reset-password HTTP/1.1
Host: api.example.com
Content-Type: application/json

"token_string_here"
```

6. Ответ при успехе

   - Код ответа: `200 OK`.
   - Формат: `application/json`.

```json
{
  "message": "string"
}
```

   Описание:
   - `message` (string) — подтверждение, что пароль успешно изменён (например, `"Пароль успешно изменен"`).

7. Ошибки

   - `400 Bad Request` — неверный или устаревший токен.

```json
{
  "message": "string"
}
```

   Возможные причины:
   - токен не корректен (не соответствует формату);
   - токен истёк (время жизни токена прошло);
   - токен уже был использован (одноразовый токен).


   - `404 Not Found` — пользователь, связанный с токеном, не найден в системе.


```json
{
  "message": "string"
}
```

   Возможные причины:
   - пользователь с этим токеном был удалён;
   - токен не связан с зарегистрированным пользователем.


   - `500 Internal Server Error` — ошибка сервера при обработке запроса (например, ошибка при обновлении пароля).


```json
{
  "message": "string"
}
```
   Возможные причины:
   - внутренняя ошибка сервера;
   - ошибка при записи нового пароля в базу данных.
