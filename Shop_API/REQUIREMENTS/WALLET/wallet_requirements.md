# REQ_WALLET_001 — Кошелёк пользователя

## 1. Общие положения

Это требование описывает функциональность пользовательского кошелька, включая отображение информации, пополнение баланса, просмотр истории операций и правила валидации.

## 2. Функциональные требования

### 2.1. Отображение информации

1. Система должна отображать текущий баланс кошелька в формате `999 999.99`  
   (с пробелами в качестве разделителей тысяч и точкой в качестве десятичного разделителя).
2. Система должна отображать общее количество совершённых операций по кошельку.  
   Это число не зависит от применённых фильтров.

### 2.2. История операций

1. Пользователь должен иметь возможность просматривать историю операций (пополнения и списания).
2. Пользователь должен иметь возможность фильтровать операции по типу:
   - "Все операции";
   - "Пополнения";
   - "Списания".
3. При изменении фильтра система должна возвращать пользователя на первую страницу истории.  
4. Пользователь должен иметь возможность настраивать количество записей на странице: `5`, `10`, `15`, `20`, `50`, `100`.  
   - По умолчанию — `10`;
   - При изменении количества записей на странице система должна возвращать пользователя на первую страницу истории.

5. При превышении количества записей на странице должна отображаться пагинация с:
   - кнопкой **"След."**;
   - кнопкой **"Пред."**;
   - информацией о текущей странице (например: "Страница 2 из 5").
6. Особенности пагинации:
   - на первой странице кнопка **"Пред."** должна быть неактивна;
   - на последней странице кнопка **"След."** должна быть неактивна;
   - на промежуточных страницах кнопки **"Пред."** и **"След."** должны быть активны.

### 2.3. Пополнение баланса

1. На странице кошелька должна быть форма пополнения, содержащая:
   - поле **"Сумма пополнения"** (обязательное) со значением по умолчанию: 1;
   - поле **"Комментарий"** (необязательное) с плейсхолдером "Например, пополнение для теста";
   - кнопку **"Пополнить"**.
2. При успешном пополнении система должна:
   - увеличить баланс на указанную сумму;
   - добавить запись в историю с типом "Пополнение".

## 3. Валидация формы пополнения

### 3.1. Валидация суммы

1. Поле **"Сумма пополнения"** обязательно для заполнения.
2. Сумма должна быть строго больше `0`.
3. Вод суммы доступен как целым так и дробным значением.
4. Сумма не может начинаться с десятичного разделителя (например, `.50` — недопустимо).  
   Минимальный формат — одна цифра до разделителя (например: `5`, `5.0`).
5. Поддерживается точность до двух знаков после точки.
6. При вводе более двух знаков после точки (например, `123.133`) система должна:
   - не изменять введённое значение автоматически;
   - отображать подсказку о лимите знаков;
   - предлагать варианты округления (например, `123.13`, `123.14`) как рекомендацию;
   - требовать, чтобы пользователь самостоятельно исправил значение.
7. Корректно введённые значения должны приводиться к формату с двумя знаками:
   - `100` → `100.00`;
   - `100,5` → `100.50`.
8. Минимальная сумма пополнения - **0.01** руб.
9. Максимальная сумма пополнения — **1 000 000.00** руб.

### 3.2. Валидация комментария

1. Поле **"Комментарий"** является необязательным.
2. Поддерживается ввод произвольного текста (буквы, цифры, спецсимволы).
3. Максимальная длина комментария — **255 символов**.

## 4. Обработка ошибок и доступность

### 4.1. Валидация

1. Все ошибки валидации обрабатываются на клиенте.
2. При ошибке система должна:
   - отображать понятное сообщение рядом с полем;
   - не выполнять операцию пополнения;
   - не менять текущий баланс.

### 4.2. Технические сбои

1. При техническом сбое (ошибка сети, недоступность сервера) система должна:
   - перехватывать ошибку;
   - отображать сообщение:  
     **"Произошла ошибка при обработке запроса. Пожалуйста, попробуйте позже."**.

### 4.3. Доступность (Accessibility)

1. Поля ввода должны иметь плейсхолдеры:
   - для суммы: "Например, 500";
   - для комментария: "Например, пополнение для теста".
2. Сообщения об ошибках должны быть связаны с полями через атрибуты `aria-describedby`.
3. Все интерактивные элементы должны:
   - быть доступны с клавиатуры;
   - иметь видимый фокус.

## 5. Дополнительные сценарии

### 5.1. Оплата корзины

1. При оплате корзины с кошелька сумма заказа списывается с текущего баланса.
2. Если средств недостаточно:
   - оплата не выполняется;
   - отображается сообщение: **"Недостаточно средств на балансе"**.
3. При успешной оплате в истории создаётся запись типа "Списание".
