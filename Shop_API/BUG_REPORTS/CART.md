# BUG_CART_001 (Исправлен)

## Заголовок
Кнопка "Заказать" смещается за пределы формы корзины при длинной общей сумме заказа

## Описание

При отображении в корзине общей суммы заказа ("Итог"), если строка с суммой становится слишком длинной (около 24 символов, включая цифры, пробелы-разделители тысяч и два знака после запятой), нарушается верстка:
- кнопка "Заказать" смещается вправо за пределы формы
- страница расширяется по ширине, появляется горизонтальная прокрутка

Это мешает пользователю увидеть и нажать кнопку "Заказать"

## Предусловия

1. В системе существует авторизованный пользователь t_user 
2. Пользователь находится в раделе "Товары"
3. В корзине есть хотя бы один товар с суммой на 24+ символа

## Шаги воспроизведения

1. Добавить в корзину товар с суммой на 24+ символа
2. Перейти в корзину

## Ожидаемый результат

1. Общая сумма заказа отображается полностью в рамках макета
2. Кнопка "Заказать" остаётся на своём месте в пределах формы
3. Не появляется горизонтальная прокрутка (страница не растягивается по ширине)

## Фактический результат

1. Кнопка "Заказать" смещается вправо за пределы формы
2. Макет корзины расширяется по ширине (появляется горизонтальная прокрутка)
3. Интерфейс выглядит поломанным, кнопка "Заказать" частично уходит за видимую область

## Серьёзность
Major

## Приоритет
High

## Окружение

- ОС: macOS 26.2
- Браузер: Comet (сборка на основе Perplexity)
  - Версия Comet: 148.0.7778.257 (официальная сборка)
  - Номер сборки Perplexity: 1743
- URL: http://localhost:3001

# BUG_CART_002 (Исправлен)

## Заголовок
Перекрытие элементов: сумма по позиции в корзине перекрывается кнопками управления товаром

## Описание

При отображении суммы по одной позиции в корзине (цена * количество), если сумма становится слишком длинной (более ~9 символов, включая цифры, пробелы и два знака после запятой), нарушается верстка:
- текст суммы заезжает под кнопки изменения количества (-, +) и кнопку удаления позиции;
- сумма частично перекрывается элементами управления.

Это ухудшает читаемость суммы и мешает пользователю понять стоимость позиции.

## Предусловия

1. В системе существует авторизованный пользователь t_user .
2. Пользователь находится в разделе "Товары"
3. В корзине есть хотя бы один товар с суммой на 9+ символов

## Шаги воспроизведения

1. Добавить товар с суммой на 9+ символов в корзину
2. Перейти в корзину

## Ожидаемый результат

1. Сумма по позиции отображается полностью
2. Текст суммы не заезжает под кнопки -, + и кнопку удаления позиции
3. Все элементы (сумма, количество, кнопки) остаются читабельными и доступными

## Фактический результат

1. Сумма по позиции заезжает под кнопки -, + и кнопку удаления
2. Часть текста суммы перекрывается элементами управления
3. Визуально верстка выглядит сломанной, информация по цене читается плохо

## Серьёзность
Major

## Приоритет
High

  - Версия Comet: 148.0.7778.257 (официальная сборка)
  - Номер сборки Perplexity: 1743
