# BUG_AUTH_REGISTER_001

## Заголовок
Нарушение консистентности валидации полей "Логин" и "Пароль" при превышении максимальной длины

## Описание
В форме регистрации поведение полей ввода отличается при превышении допустимой длины

Поле "Логин" не позволяет ввести больше 32 символов и не показывает предупреждение, а поле "Пароль" позволяет ввести больше 64 символов и показывает предупреждение

## Предусловия
1. Пользователь не авторизован
2. Пользователь находится на странице регистрации http://localhost:3001/login

## Шаги воспроизведения
1. Перейти в форму регистрации
2. В поле "Логин" ввести строку длиной более 32 символов
3. Проверить поведение поля "Логин"
4. В поле "Пароль" ввести строку длиной более 64 символов
5. Проверить поведение поля "Пароль"

## Ожидаемый результат
Поведение полей должно быть консистентным:
- либо оба поля ограничивают ввод на максимальной длине без предупреждения,
- либо оба поля позволяют ввод сверх лимита с отображением сообщения о превышении

## Фактический результат
1. Поле "Логин" не принимает больше 32 символов и не отображает предупреждение
2. Поле "Пароль" принимает больше 64 символов и отображает предупреждение

## Серьёзность
Minor

## Приоритет
Medium

## Окружение
- Shop App: 2.1.0
- ОС: macOS 26.2
- Браузер: Comet (сборка на основе Perplexity)
  - Версия Comet: 148.0.7778.257 (официальная сборка)
  - Номер сборки Perplexity: 1743
- URL: http://localhost:3001

# BUG_AUTH_LOGIN_001

## Заголовок
Блокировка учётной записи при многократном входе с верными учетными данными.

## Описание
При 5-кратном входе и выходе из системы в течение минуты учётная запись пользователя временно блокируется, хотя логин и пароль введены корректно и ошибок авторизации нет.

## Предусловия
1. Пользователь не авторизован
2. Пользователь находится на странице входа http://localhost:3001/login

## Тестовые данные
1. Логин: `t_user`
2. Пароль: `User123!`

## Шаги воспроизведения
1. Перейти на страницу входа в систему
2. В поле "Логин" ввести значение из тестовых данных
3. В поле "Пароль" ввести значение из тестовых данных
4. Нажать кнопку "Войти"
5. Нажать кнопку "Выйти"
6. Повторить шаги 2-5 5 раз

## Ожидаемый результат
1. Пользователь успешно авторизован
2. Пользователь попадает в раздел "Товары"

## Фактический результат
1. После 5-кратного входа и выхода из системы за короткое время попытка очередного входа завершается блокировкой учётной записи.
2. Пользователь не может авторизоваться в течение периода блокировки.
3. Отображается сообщение: "Превышено количество попыток входа. Пожалуйста, попробуйте позже".

## Серьёзность
Minor

## Приоритет
Medium

## Окружение
- Shop App: 2.1.0
- ОС: macOS 26.2
- Браузер: Comet (сборка на основе Perplexity)
  - Версия Comet: 148.0.7778.257 (официальная сборка)
  - Номер сборки Perplexity: 1743
- URL: http://localhost:3001
