# BUG_PROFILE_001 

## Заголовок
Ложное срабатывание менеджера паролей браузера при отмене смены пароля

## Описание

После ввода любых данных в поля формы смены пароля и нажатия кнопки "Отмена", и перехода в любой другой раздел магазина, браузер предлагает сохранить введённые данные как пароль. Это происходит даже если поля были программно очищены после нажатия кнопки.  

Из-за этого:
- Менеджер паролей предлагает сохранить потенциально некорректные или мусорные данные.
- Пользователь может по ошибке сохранить неверный пароль, что приведёт к проблемам при следующей попытке входа.
- Создаётся ложное впечатление, что в системе был создан новый пользователь или успешно изменён пароль.

## Предусловия

1. В системе существует авторизованный пользователь t_user

## Шаги воспроизведения

1. Перейти в раздел "Профиль" → "Изменить пароль".
2. Ввести любые данные (например, `abc`) в любое из полей: "Текущий пароль", "Новый пароль" или "Подтверждение".
3. Нажать кнопку "Отмена".
4. Перейти в любой другой раздел системы (например, в раздел "Товары").

## Ожидаемый результат

1. Данные из полей формы не сохраняются.
2. Менеджер паролей браузера не активируется и не предлагает сохранить пароль.

## Фактический результат

1. После перехода в другой раздел появляется всплывающее окно менеджера паролей браузера.
2. Менеджер паролей предлагает сохранить введённые данные (даже если это был невалидный набор символов).
3. Эффект наблюдается даже если заполнить только одно поле из трёх.
4. Эффект не наблюдается, если сначала вручную стереть введённые данные, а затем нажать "Отмена".
