# BUG_PROD_001

## Название
Залипающий ноль в поле "Цена" при создании товара

## Описание
В форме создания товара поле "Цена" изначально заполнено значением по умолчанию 10. При попытке изменить это значение:
 - если стереть его, в поле автоматически появляется 0
 - если вводить новое число, когда в поле отображается 0, ноль остаётся впереди (например, 0123)

Поведение поля выглядит некорректно и сбивает пользователя с толку, несмотря на то, что при сохранении товара значение цены обрабатывается корректно.

## Предусловия
1. В системе существуют пользователи:
   - tadmin (роль admin)
   - tseller (роль seller)
2. Пользователь авторизован под логином tadmin или tseller и находится в разделе "Товары"

## Шаги воспроизведения
1. Нажать кнопку "Новый товар"
2. Обратить внимание, что поле "Цена" по умолчанию содержит значение 10
3. Полностью стереть значение в поле "Цена" клавишами Backspace/Delete.
4. Наблюдать, какое значение отображается в поле после очистки.
5. Не удаляя появившийся 0, ввести новое число, например 123.

## Ожидаемый результат
1. После очистки значения по умолчанию поле "Цена" либо:
 - остаётся пустым, либо
 - сразу принимает вводимое значение без промежуточного 0
2. При вводе нового числа значение отображается без ведущего нуля (например, 123)

## Фактический результат
1. После очистки значения 10 в поле "Цена" автоматически отображается 0
2. При вводе нового числа в поле, содержащем 0, значение отображается с ведущим нулём, например 0123
3. Визуально создаётся ощущение, что ноль является частью вводимой цены, что может вводить пользователя в заблуждение
