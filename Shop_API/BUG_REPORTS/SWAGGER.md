# BUG_SWAGGER_001

## Название
Документация Swagger указывает возможность зарегистрировать пользователя с заглавными буквами, но API и UI разрешают только строчные буквы (a-z)

## Описание
1. В Swagger документации указано, что можно зарегистрировать пользователя с заглавными буквами (например, UsEr)
2. При попытке зарегистрировать пользователя с заглавными буквами через API возвращается ошибка
3. Через UI также указано, что можно зарегистрировать пользователя только со строчными буквами (например, user)

## Шаги воспроизведения
1. Открыть Swagger документацию
2. Найти endpoint для регистрации пользователя (POST /register)
3. Посмотреть поле username в документации (раздел schema)
4. Увидеть, что в Swagger указано, что можно использовать значение с заглавными буквами
5. Открыть UI для регистрации пользователя
6. Увидеть, что в UI указано требование — только со строчными буквами (например, "user")
7. Зафиксировать несоответствие между Swagger и UI

## Ожидаемый результат
Документация Swagger должна точно указывать:
 - Что username должен состоять только из строчных букв
 - Ограничения на валидацию ([a-z], минимум 1 символ)
 - Примеры корректных значений (например, user, anna, john)

## Фактический результат
1. В документации Swagger указано, что можно зарегистрировать пользователя с заглавными буквами, но:
 - API возвращает ошибку: 
 {
  "code": 400,
  "message": "Логин должен содержать только английские буквы (a-z)"
}
 - UI также запрещает регистрацию пользователя с заглавными буквами
