# BUG_API_PROD_001 (Исправлено)

## Заголовок
Нарушение консистентности: API позволяет создать товар с ценой 0, но UI блокирует это действие

## Описание
При создании товара минимальная цена валидируется по-разному на уровне API и UI:

- В UI минимальная допустимая цена товара — 1 (значение 0 отклоняется)
- В API минимальная допустимая цена товара — 0 (значение 0 принимается и сохраняется)

Из-за этого:
- через UI (ADMIN/SELLER) товар с ценой 0 создать нельзя
- через API — создать товар можно
- нарушается единообразие бизнес-логики: минимальная цена товара различается в UI и API

## Предусловия
- Открыт REST клиент Insomnia (или любой аналог)
- В системе существуют пользователи с ролями ADMIN/SELLER
- Получен токен для обращения к API от имени ADMIN/SELLER
- Доступен endpoint создания товара в API - `POST http://localhost:3000/products`
- Установлены заголовки:
 - accept: application/json
 - Content-Type: application/json
 - Authorization: (Bearer [TOKEN])

## Шаги воспроизведения
1. Отправить запрос с телом:

```
{
  "name": "[название товара]",
  "price": 0,
  "available": true
}
```

2. Проверить ответ API
3. Убедиться, что товар с ценой 0 создан запросом `GET http://localhost:3000/products`

## Ожидаемый результат

1. Товар не создаётся 
2. Тело ответа:

```
{
  "code": 400,
  "message": "price must be at least 1"
}
```

3. В списке товаров GET запроса нет товара с "name: <название товара>"

## Фактический результат

1. Товар создался
2. Тело ответа:

```
  {
	"id": "<generated_id>",
	"name": "<название товара>",
	"price": 0,
	"available": true
}
```

 - Код: 201 Created

3. В списке товаров GET запроса существует товар с "name: <название товара>"
