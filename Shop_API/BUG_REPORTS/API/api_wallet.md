# BUG_API_WALLET_001

## Заголовок
Нарушение консистентности: API позволяет пополнить кошелёк на сумму меньше 1 рубля, но фронтенд блокирует это действие

## Описание
При пополнении кошелька минимальная сумма валидируется по-разному на уровне API и фронтенда:

- На фронтенде минимальная допустимая сумма пополнения — 1 рубль, значения меньше 1 отклоняются.
- В API минимальная допустимая сумма пополнения — 0.01 рубля, значения меньше 1 принимаются и обрабатываются.

Из-за этого:
- через фронтенд пользователь не может пополнить кошелёк на сумму меньше 1 рубля;
- через API пополнение на сумму меньше 1 рубля возможно;
- нарушается единообразие бизнес-логики: минимальная сумма пополнения различается на фронтенде и API.

## Предусловия
1. Открыт REST-клиент Insomnia или любой аналог.
2. В системе существует хотя бы один пользователь.
3. Получен токен для обращения к API от имени существующего пользователя.
4. Доступен endpoint пополнения кошелька в API — `PATCH http://localhost:3000/wallet`.
5. Установлены заголовки:
   - `accept: application/json`
   - `Content-Type: application/json`
   - `Authorization: Bearer [TOKEN]`

## Шаги воспроизведения
1. Отправить запрос с телом:

```json
{
  "operation": "DEPOSIT",
  "amount": 0.01,
  "description": "[комментарий к пополнению]"
}
```

2. Проверить ответ API.
3. Убедиться, что баланс кошелька увеличился на `0.01` запросом `GET http://localhost:3000/wallet`.
4. Проверить историю операций кошелька запросом `GET http://localhost:3000/wallet/transactions?offset=0`.

## Ожидаемый результат
1. Пополнение не выполняется.
2. Тело ответа:

```json
{
  "code": 400,
  "message": "amount must be at least 1"
}
```

3. В данных кошелька нет записи о пополнении на сумму меньше 1 рубля.

## Фактический результат
1. Пополнение на сумму `0.01` выполняется.
2. Тело ответа:

```json
{
  "id": "<<идентификатор пользователя>>",
  "balance": 0.01
}
```

- Код: `200 OK`

3. В данных кошелька существует операция пополнения на сумму меньше 1 рубля.
