# TEST_PLAN_WALLET_UI — План UI-тестирования кошелька

## 1. Общая информация

- **Модуль:** Кошелёк пользователя (`REQ_WALLET_001`).
- **Тип тестирования:** UI (без проверки API и backend-логики).
- **Цель:** Проверить корректность отображения баланса, истории операций и формы пополнения, а также валидации и доступности интерфейса.

## 2. Команда и роли

- **Автор тест-плана / тест-кейсов:** Anka.
- **Исполнитель UI-тестов:** Anka.
- **Ответственный за фиксы:** Alebymars.
- **Ответственный за приоритизацию дефектов:** Anka.

## 3. Окружение

- **Приложение:** Shop App 2.1.0.
- **ОС:** macOS 26.2.
- **Браузер:** Comet 148.0.7778.257 (сборка на основе Perplexity).
- **URL:** `http://localhost:3001`.


## 4. Виды тестирования

- Функциональное UI-тестирование.
- Позитивное / негативное тестирование.
- Тестирование валидации форм.
- Тестирование доступности (клавиатура, фокус, aria-связи).
- Лёгкая регрессия ключевых сценариев кошелька после изменений.

## 5. Техники тест-дизайна

- Эквивалентное разбиение (валидные/невалидные суммы, длина комментария).
- Анализ граничных значений (0, минимальная >0, максимум, чуть выше/ниже границ).
- Папарное тестирование (фильтры + пагинация + количество записей).
- Тестирование состояний и переходов (изменение фильтра, изменение размера страницы, переход по страницам).

## 6. Приоритеты тестирования

- **High:**
  - Отображение баланса кошелька.
  - Пополнение баланса (валидные и невалидные суммы).
  - Ошибки валидации суммы.
  - Оплата корзины с кошелька и проверка сообщения о недостатке средств.
- **Medium:**
  - История операций (список, фильтры, пагинация).
  - Валидация комментария.
  - Тост-уведомления.
- **Low:**
  - Тексты подсказок и плейсхолдеров.
  - Мелкие визуальные несоответствия, не влияющие на функциональность.

## 7. Критерии начала тестирования

- Реализован и доступен функционал кошелька согласно `REQ_WALLET_001`.
- Доступны тестовые учётные записи.

## 8. Критерии окончания тестирования

- Выполнены все тест-кейсы с приоритетом **High** и **Medium**.
- Отсутствуют открытые дефекты с критичностью **Blocker / Critical** по кошельку.
- Количество дефектов с критичностью **Major** принято командой как допустимое.
- Основные пользовательские сценарии кошелька (просмотр баланса, просмотр истории, пополнение, оплата корзины) работают стабильно.

## 9. Инструменты

- Трекер задач / багов: Git (Markdown) .
- Хранение тест-кейсов: Git (Markdown).
- Браузерные DevTools для проверки верстки и состояний.
- Средства записи экрана/скриншотов для фиксации дефектов.

## 10. Тестовая документация

- Требования: `REQ_WALLET_001 — Кошелёк пользователя`.
- Тест-кейсы по кошельку: `WALLET_POS_*`, `WALLET_NEG_*`.
- Матрица трассировки: требования ↔ тест-кейсы по кошельку.
- Баг-репорты по модулю кошелька.
- Краткий отчёт по результатам прогона (количество кейсов, % пройдено, дефекты по severity).

## 11. Метрики тестирования

- Общее количество тест-кейсов по кошельку.
- Покрытие требований тест-кейсами (кол-во требований из `REQ_WALLET_001`, имеющих хотя бы один тест).
- Количество найденных дефектов по severity (Blocker/Critical/Major/Minor).
- % успешно пройденных тест-кейсов.
- Количество регрессионных дефектов при изменениях в кошельке.
