# PROD_FILTER_POS_001 

## Название
Успешная фильтрация товаров по доступности: значение "Все"

## Связанные требования
1. REQ_PRODUCT_006 — Фильтрация товаров по доступности

## Предусловия
1. Пользователь tuser авторизован и находится в разделе "Товары"
2. В системе существуют товары с разными значениями:
 - хотя бы один товар с признаком "В наличии"
 - хотя бы один товар с признаком "Нет в наличии"

## Шаги воспроизведения
1. В фильтре "Доступность товара" выбрать "Все"

## Ожидаемый результат
1. Список товаров обновляется
2. В списке отображаются товары:
 - с признаком "В наличии"
 - с признаком "Нет в наличии"
3. Все существующие товары отображаются в списке

## Статус
Active

## Приоритет
Medium

## Окружение
- ОС: macOS 26.2
- Браузер: Comet
- URL: http://localhost:3001

## PROD_PAGE_SIZE_POS_001

## Название
Успешное изменение количества товаров на странице

## Связанные требования
1. REQ_PRODUCT_005 — Настройка количества отображаемых товаров

## Предусловия
1. Пользователь tuser авторизован и находится в разделе "Товары"
2. В системе существует больше 100 товаров

## Тестовые данные
1. Значения количества товаров на странице: 5, 10, 15, 20, 50, 100

## Шаги воспроизведения
1. Для каждого значения из списка тестовых данных последовательно выполнить:
 - В выпадающем списке "На странице" выбрать соответствующее значение
 - Убедиться, что список товаров обновился

## Ожидаемый результат
1. Для каждого выбранного значения количества товаров на странице:
 - На странице отображается количество товаров, соответствующее выбранному значению
 - Если товаров в системе больше выбранного значения, отображается ровно выбранное количество товаров
2. Если товаров в системе меньше выбранного значения, отображаются все доступные товары

## Статус
Active

## Приоритет
Medium

## Окружение
- ОС: macOS 26.2
- Браузер: Comet
- URL: http://localhost:3001
