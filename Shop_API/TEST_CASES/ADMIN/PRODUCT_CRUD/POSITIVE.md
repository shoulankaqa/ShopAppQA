# PROD_CREATE_POS_001 

## Название
Успешное создание товара (admin/seller)

## Связанные требования
1. REQ_PRODUCT_002 - Создание товара (admin/seller)

## Предусловия
1. В системе существуют пользователи:
   - tadmin (роль admin)
   - tseller (роль seller)
2. Пользователь авторизован под логином tadmin или tseller
   и находится в разделе "Товары"

## Тестовые данные
1. Наименование товара: строка с названием (например, Test product)
2. Цена: целое положительное число от 1 (например, 100)
3. Наличие товара: валидное значение (например, чекбокс «В наличии» установлен)

## Шаги воспроизведения
1. Нажать кнопку "Новый товар"
2. Заполнить поля формы данными из тестовых данных
3. Нажать кнопку "Сохранить"

## Ожидаемый результат
1. Товар успешно создаётся
2. Пользователь остаётся авторизованным
3. Новый товар отображается в списке товаров с указанными наименованием, ценой и выбранным признаком наличия

## Статус
Active

## Приоритет
High

## Окружение
- ОС: macOS 26.2
- Браузер: Comet
- URL: http://localhost:3001

## Дополнительно
Тест должен быть выполнен дважды:
 - под пользователем tadmin (admin)
 - под пользователем tseller (seller)

# PROD_UPDATE_POS_001

## Название
Успешное редактирование товара (admin/seller)

## Связанные требования
1. REQ_PRODUCT_003 — Редактированиe товара (admin/seller)

## Предусловия
1. В системе существуют пользователи:
   - tadmin (роль admin)
   - tseller (роль seller)
2. Пользователь авторизован под логином tadmin или tseller и находится в разделе "Товары"
3. В системе существует хотя бы один товар

## Тестовые данные
1. Новое наименование товара: Updated product
2. Новая цена: 200
3. Новое наличие товара: установить противоположное текущему значению чекбокса

2. Нажать кнопку "Редактировать"
3. Изменить данные товара значениями из тестовых данных
4. Нажать кнопку "Сохранить"

1. Изменения товара успешно сохраняются
2. Пользователь остаётся авторизованным
3. В списке товаров отображаются обновлённые данные:
 - наименование: Updated product
 - цена: 200
