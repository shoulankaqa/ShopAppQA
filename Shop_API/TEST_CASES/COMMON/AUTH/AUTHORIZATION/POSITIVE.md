# AUTH_LOGIN_POS_001 

## Название
Успешная авторизация зарегистрированного пользователя

## Связанные требования
1. REQ_AUTH_002 - требования к авторизации

## Предусловия
1. В системе существует зарегистрированный пользователь t_user 
2. Пользователь t_user не авторизован

## Тестовые данные
1. Логин: t_user 
2. Пароль: User123!

## Шаги воспроизведения
1. Открыть страницу "Вход в панель" http://localhost:3001/login
2. Ввести логин из тестовых данных
3. Ввести пароль из тестовых данных
4. Нажать кнопку "Войти"

## Ожидаемый результат
1. Пользователь успешно авторизован
2. Происходит редирект в раздел "Товары"
3. Отображается сообщение: "Успех. Авторизовались под: t_user"

# AUTH_POS_002

## Название
Успешное снятие временной блокировки и возможность повторной авторизации через 15 минут

## Связанные требования
1. REQ_AUTH_002 — Авторизация и выход из системы (п. 2.2.5)

## Предусловия
1. В системе существует пользователь `t_user` 
2. Пользователь `t_user` ранее выполнил 5 неверных попыток входа подряд и получил временную блокировку
3. Открыта форма авторизации

## Тестовые данные
1. Логин: t_user 
2. Пароль: User123!

## Шаги воспроизведения
1. Дождаться истечения 15 минут с момента последней неверной попытки входа пользователя `t_user`
2. Ввести логин пользователя `t_user` и корректный пароль
3. Нажать кнопку "Войти"

## Ожидаемый результат
1. Временная блокировка для пользователя `t_user` снимается по истечении 15 минут
2. Пользователь `t_user` успешно авторизуется при вводе корректного пароля
3. Сообщение о временной блокировке `"Превышено количество попыток входа. Пожалуйста, попробуйте позже."` не отображается после истечения периода блокировки

## Статус
Active

## Приоритет
High

- Браузер: Comet (сборка на основе Perplexity)
  - Версия Comet: 148.0.7778.257 (официальная сборка)
  - Номер сборки Perplexity: 1743
- URL: http://localhost:3001

# AUTH_POS_003

## Название
Успешная авторизация до достижения лимита неверных попыток

## Связанные требования
1. REQ_AUTH_002 — Авторизация и выход из системы (п. 2.2.5)

## Предусловия
1. В системе существует пользователь `t_user`
2. Пользователь `t_user` не заблокирован
3. Открыта форма авторизации

## Тестовые данные
1. Логин: t_user 
2. Корректный пароль: User123!
3. Неверный пароль: WrongPassword123456!

## Шаги воспроизведения
1. Ввести логин пользователя `t_user` и неверный пароль
2. Нажать кнопку "Войти"
3. Повторить шаги 1–2 ещё 3 раза (всего 4 неверные попытки в течение 15 минут)
4. На 5-й попытке ввести логин пользователя `t_user` и корректный пароль
5. Нажать кнопку "Войти"

## Ожидаемый результат
1. После 4 неверных попыток входа временная блокировка не активируется
2. На 5-й попытке при вводе корректного пароля пользователь `t_user` успешно авторизуется
3. Сообщение `"Превышено количество попыток входа. Пожалуйста, попробуйте позже."` не отображается, так как лимит неверных попыток не исчерпан
