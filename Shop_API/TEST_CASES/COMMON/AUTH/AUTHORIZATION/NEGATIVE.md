# AUTH_LOGIN_NEG_001 

## Название
Авторизация несуществующего пользователя

## Связанные требования
1. REQ_AUTH_002 - требования к авторизации

## Предусловия
1. Пользователь nonexistuser не зарегистрирован в системе
2. Пользователь nonexistuser не авторизован

## Тестовые данные
1. Логин: nonexistuser 
2. Пароль: NoLogin123! 

## Шаги воспроизведения
1. Открыть страницу "Вход в панель" http://localhost:3001/login
2. Ввести логин из тестовых данных
3. Ввести пароль из тестовых данных
4. Нажать кнопку "Войти"

## Ожидаемый результат
1. Авторизация не выполняется
2. Пользователь остается на странице "Вход в панель"
3. Отображается сообщение: "Неверные учетные данные"

# AUTH_NEG_003

## Название
Временная блокировка после 5 неверных попыток входа

## Связанные требования
1. REQ_AUTH_002 — Авторизация и выход из системы (п. 2.2.5)

## Предусловия
1. В системе существует пользователь `t_user`
2. Пользователь `t_user` не заблокирован и может авторизоваться
3. Открыта форма авторизации

## Шаги воспроизведения
1. Ввести логин пользователя `t_user` и неверный пароль
2. Нажать кнопку "Войти"
3. Убедиться, что авторизация неуспешна
4. Повторить шаги 1–3 ещё 4 раза в течение 15 минут (всего 5 неверных попыток подряд)
5. Выполнить 6-ю попытку входа с неверным паролем

## Ожидаемый результат
1. После выполнения 5-й неверной попытки дальнейшие попытки авторизации временно блокируются
2. При выполнении 6-й попытки входа система не выполняет аутентификацию пользователя
3. Отображается сообщение о временной блокировке:  
   `"Превышено количество попыток входа. Пожалуйста, попробуйте позже."`
4. Вход в систему для пользователя `t_user` недоступен до истечения времени блокировки

## Статус
Active

## Приоритет
High

## Окружение
- Shop App: 2.1.0
- ОС: macOS 26.2
- Браузер: Comet (сборка на основе Perplexity)
  - Версия Comet: 148.0.7778.257 (официальная сборка)
  - Номер сборки Perplexity: 1743
- URL: `http://localhost:3001`
