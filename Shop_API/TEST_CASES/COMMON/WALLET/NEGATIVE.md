# WALLET_NEG_001

## Название
Пополнение баланса кошелька больше максимально допустимой суммы (MAX + 1)

## Связанные требования
1. REQ_WALLET_001 - Кошелёк пользователя (п. 3.1.9)

## Предусловия
1. В системе существует авторизованный пользователь t_user

## Тестовые данные
1. Сумма пополнения: `1000001` (MAX + 1)

## Шаги воспроизведения
1. Перейти в раздел "Кошелёк"
2. Перейти к форме пополнения кошелька
3. Попытаться ввести в поле "Сумма пополнения" значение из тестовых данных

## Ожидаемый результат
1. Система блокирует ввод значения, превышающего 1 000 000.

# WALLET_NEG_003

## Название
Отказ в списании средств с кошелька при недостаточном балансе

## Связанные требования
1. REQ_WALLET_001 — Кошелёк пользователя (п. 5.1.2)

## Предусловия
1. В системе существует пользователь `t_user`
2. Пользователь `t_user` авторизован и находится в разделе "Товары"
3. В корзине пользователя находится хотя бы один товар
4. В кошельке пользователя находится недостаточная сумма для оплаты корзины

## Шаги воспроизведения
1. Перейти в раздел "Кошелёк"
2. Запомнить текущий баланс пользователя
3. Перейти в корзину
4. Нажать кнопку "Заказать"

## Ожидаемый результат
1. Операция оплаты не выполняется
2. Баланс кошелька остаётся без изменений
3. Отображается сообщение: `Недостаточно средств на балансе`
4. В истории операций не создаётся запись с типом `"Списание"`
