# WALLET_POS_001

## Название
Успешное отображение баланса кошелька

## Связанные требования
1. REQ_WALLET_001 - Кошелёк пользователя (п. 2.1.1)

## Предусловия
1. В системе существует авторизованный пользователь t_user

## Шаги воспроизведения
1. Перейти в раздел "Кошелёк"
2. Проверить отображение баланса

## Ожидаемый результат
1. Баланс отображается в формате `999 999.99`— с пробелами - разделителями тысяч и точкой в качестве десятичного разделителя.

# WALLET_POS_010

## Название
Корректное состояние кнопок пагинации "Пред" и "След" в истории операций кошелька

## Связанные требования
1. REQ_WALLET_001 - Кошелёк пользователя (п. 2.2.5)

## Предусловия
1. В системе существует авторизованный пользователь t_user
2. Кошелёк содержит операций больше, чем помещается на одной странице
3. Для проверки доступны первая, промежуточная и последняя страницы истории

## Шаги воспроизведения
1. Перейти в раздел "Кошелёк"
2. Открыть первую страницу истории операций
3. Проверить состояние кнопок пагинации
4. Перейти на промежуточную страницу истории операций
5. Проверить состояние кнопок пагинации
6. Перейти на последнюю страницу истории операций
7. Проверить состояние кнопок пагинации

## Ожидаемый результат
1. На первой странице кнопка "Пред" неактивна
2. На первой странице кнопка "След" активна
3. На последней странице кнопка "След" неактивна
4. На последней странице кнопка "Пред" активна
5. На промежуточной странице кнопки "Пред" и "След" активны

Успешное пополнение баланса кошелька больше минимально допустимой суммы (MIN + 1)

## Связанные требования
1. REQ_WALLET_001 - Кошелёк пользователя (п. 3.1.8)

## Предусловия
1. В системе существует авторизованный пользователь t_user

## Тестовые данные
1. Сумма пополнения: `2` (MIN + 1)
2. Комментарий: например, `Тест`

## Шаги воспроизведения
1. Перейти в раздел "Кошелёк"
2. Перейти к форме пополнения кошелька
3. Ввести в поле "Сумма пополнения" значение из тестовых данных
4. Ввести в поле "Комментарий" значение из тестовых данных
5. Нажать кнопку "Пополнить"

## Ожидаемый результат
1. Пополнение выполнено успешно
2. Баланс кошелька увеличился на 2
3. В истории появилась запись с типом "Пополнение" с указанной суммой и комментарием

# WALLET_POS_025

## Название
Успешное списание суммы корзины с кошелька

## Связанные требования
1. REQ_WALLET_001 — Кошелёк пользователя (п. 5.1.1)

## Предусловия
1. В системе существует пользователь `t_user`
2. Пользователь `t_user` авторизован и находится в разделе "Товары"
3. В корзине пользователя находится хотя бы один товар
4. В кошельке пользователя находится достаточная сумма для оплаты корзины

## Шаги воспроизведения
1. Перейти в раздел "Кошелёк"
2. Запомнить текущий баланс пользователя
3. Перейти в корзину
4. Нажать кнопку "Заказать"

## Ожидаемый результат
1. Баланс кошелька уменьшается на сумму, равную итоговой стоимости корзины
2. Отображается сообщение: `Вы оплатили корзину на <сумма корзины>`
3. В истории операций кошелька создаётся запись с типом `"Списание"`, содержащая:
   - сумму операции, равную общей стоимости оплаченной корзины;
   - дату и время операции;
   - комментарий: `Оплата корзины`
