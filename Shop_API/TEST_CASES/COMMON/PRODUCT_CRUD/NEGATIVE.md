# PROD_CREATE_NEG_001 

## Название
Создание товара без названия (admin/seller)

## Связанные требования
1. REQ_PRODUCT_002 - Создание товара (admin/seller)

## Предусловия
1. В системе существуют пользователи:
   - tadmin (роль admin)
   - tseller (роль seller)
2. Пользователь авторизован под логином tadmin или tseller и находится в разделе "Товары"

## Тестовые данные
1. Цена: целое положительное число от 1 (например, 100)
2. Наличие товара: валидное значение (например, чекбокс «В наличии» установлен)

## Шаги воспроизведения
1. Нажать кнопку "Новый товар"
2. Оставить поле "Наименование" пустым
3. Заполнить полe "Цена"
4. Отметить чекбокс наличия товара
5. Нажать кнопку "Сохранить"

## Ожидаемый результат
1. Товар не создается
2. Под полем "Наименование" отображается сообщение "Заполните это поле"
3. Пользователь остаётся на форме создания товара, введённые значения в других полях сохраняются

## Статус
Active

## Приоритет
High

## Окружение
- ОС: macOS 26.2
- Браузер: Comet
- URL: http://localhost:3001

## Дополнительно
Тест должен быть выполнен дважды:
 - под пользователем tadmin (admin)
 - под пользователем tseller (seller)

2. Цена товара: отрицательное целое значение (например, -11)
3. Наличие товара: валидное значение (например, чекбокс «В наличии» установлен)

## Шаги воспроизведения

1. Система не позволяет ввести отрицательное значение:
 - при вводе "-" в поле "Цена" значение автоматически становится 0
